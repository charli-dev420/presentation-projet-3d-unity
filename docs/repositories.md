# Repositories Covered / Repos couverts

[EN](#english) | [FR](#francais)

This showcase repository represents several real 3D, Unity, dataset, and asset-pipeline repositories. It publishes presentation material only.

## English

| Real source | Status | Role in the showcase | What is published here |
| --- | --- | --- | --- |
| [`charli-dev420/datasetvieweval`](https://github.com/charli-dev420/datasetvieweval) | Public | Dataset ReviewEval: local Windows app for evaluating and preparing Flux/Trellis-style visual datasets. | Dataset workflow, quality criteria, operator resources, proof framing. |
| [`charli-dev420/splat-facade-baker`](https://github.com/charli-dev420/splat-facade-baker) | Public | Splat Face: 2.5D route for turning visual sources, maps, or splat-style references into lightweight Unity/mobile candidates. | Product description, asset journey, expected validation, demo use cases. |
| [`charli-dev420/codextounity`](https://github.com/charli-dev420/codextounity) | Public | CodexUnity: Codex / Unity / ComfyUI prototype for generating, reviewing, normalizing, and importing assets. | Orchestration role, Unity integration framing, experimental boundaries. |
| Mob'ia / ccomf-unity | Private | Product layer around ComfyUI-style profiles, jobs, artifacts, and Unity/web/mobile clients. | Public-safe architecture, user journeys, buyer and partner positioning. |
| Local Unity / LocalAssetFactory pipeline | Private/local | Local service and Unity command loop for preflight, normalization, asset validation, and scene handoff. | Concepts, QA criteria, demo framing, validation needs. |

### Publication Rule

Public repositories can be linked. Private or local sources can be named as project areas only; no private workflows, model files, datasets, meshes, generated assets, Unity project internals, secrets, endpoints, or local paths are published here.
