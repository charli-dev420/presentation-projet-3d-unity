# Roadmap / Feuille de route

[EN](#english) | [FR](#francais)

## English

| Horizon | Objective | Public output expected |
| --- | --- | --- |
| Short term | Keep the public dossier coherent and buyer-readable. | README, one-pager, user flows, proof cards, visual index. |
| Short term | Clarify the multi-surface map. | Exact role of Dataset ReviewEval, Splat Face, CodexUnity, Mob'ia / ccomf-unity, and LocalAssetFactory. |
| Medium term | Stabilize asset validation criteria. | Public checklists and written examples for keep/fix/reject/accept decisions. |
| Medium term | Strengthen Unity import and mobile constraints. | Scoped demos and public-safe proof summaries using cleared assets. |
| Long term | Package partner evaluation. | Evaluation kit, optional NDA path, controlled demo, buyer/funder documentation. |

### Roadmap Principle

Each milestone should produce a document or proof that helps another person decide. Avoid broad demos that do not close a question.

## Francais

| Horizon | Objectif | Sortie publique attendue |
| --- | --- | --- |
| Court terme | Garder le dossier public coherent et lisible par un acheteur. | README, one-pager, flux utilisateur, cartes de preuve, index visuel. |
| Court terme | Clarifier la carte multi-surfaces. | Role exact de Dataset ReviewEval, Splat Face, CodexUnity, Mob'ia / ccomf-unity et LocalAssetFactory. |
| Moyen terme | Stabiliser les criteres de validation asset. | Checklists publiques et exemples rediges de decisions garder/corriger/rejeter/accepter. |
| Moyen terme | Renforcer import Unity et contraintes mobile. | Demos ciblees et resumes de preuve public-safe avec assets autorises. |
| Long terme | Packager l'evaluation partenaire. | Kit d'evaluation, chemin NDA optionnel, demo controlee, documentation acheteur/financeur. |

### Principe roadmap

Chaque jalon doit produire un document ou une preuve qui aide une autre personne a decider. Eviter les demos larges qui ne ferment aucune question.
