# Readiness Notes / Notes de diligence

[EN](#english) | [FR](#francais)

## English

### Readiness Items

| Item | Priority | Why it matters | Useful output |
| --- | --- | --- | --- |
| Unity validation in a disposable project | P0 | Import, prefab/scene placement, and runtime constraints make the pipeline tangible for a Unity reviewer. | Public-safe Unity validation note with accepted/rejected criteria. |
| Short ComfyUI-style live run | P0 | A controlled run gives the product layer a practical story without exposing workflow internals. | Redacted run summary with input type, output discovery, and review decision. |
| Cleared GLB or visual asset turntable | P1 | Reviewers need to see how shape, scale, maps, and materials behave on an allowed asset. | Screenshot or written turntable summary using public or NDA-cleared material. |
| Mesh and material validation | P1 | A usable asset needs bounds, triangle budget, textures, materials, and profile checks. | QA checklist result attached to a demo scenario. |
| Dataset and model license matrix | P1 | Distribution and demo rights must be clear before a public or buyer-facing proof. | License/shareability decision table. |
| Dataset quality examples | P2 | Source selection is the first quality gate and the easiest part for partners to understand. | Public-safe examples of keep, fix, reject, and manual-review decisions. |

### How To Use These Notes

These are not presented as failures. They are the practical next gates that turn a credible public dossier into buyer-grade or partner-grade proof.

### Comment lire ces notes

Ces points ne sont pas presentes comme des echecs. Ce sont les prochains gates pratiques pour transformer un dossier public credible en preuve de niveau acheteur ou partenaire.
