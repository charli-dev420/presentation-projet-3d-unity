# Validation Scenarios / Scenarios de validation

[EN](#english) | [FR](#francais)

## English

### Scenario 1 - Source Review Before Asset Work

Take five visual sources. For each one, write what it shows, whether it fits a facade or 2.5D route, what is weak, and whether it should be kept, fixed, rejected, or revisited. The output is a readable source decision set.

### Scenario 2 - Splat Face Candidate Review

Choose one source that survived review. Define the intended Unity role, expected framing, depth-card logic, texture readability, platform constraint, and first acceptance criteria. The output is a candidate review note.

### Scenario 3 - Unity Handoff Check

Prepare the handoff as if another Unity reviewer had to inspect it: name, source note, format, scale, orientation, pivot, bounds, materials, texture expectations, and scene role. The output is a Unity-facing checklist result.

### Scenario 4 - Product Workflow Review

Frame the same candidate through Mob'ia / ccomf-unity language: profile, job, artifact, state, client, and next action. The output is a product-readable workflow state.

### Scenario 5 - LocalAssetFactory Preparation

Run the candidate through local preparation language: manifest, normalization, expected import result, review status, and next action. The output is a local asset readiness note.

## Francais

### Scenario 1 - Revue Source Avant Travail Asset

Prendre cinq sources visuelles. Pour chacune, ecrire ce qu'elle montre, si elle correspond a une route facade ou 2.5D, ce qui est faible et s'il faut garder, corriger, refuser ou revoir plus tard. La sortie est un lot de decisions source lisible.

### Scenario 2 - Revue Candidat Splat Face

Choisir une source qui a passe la revue. Definir le role Unity attendu, le cadrage, la logique depth-card, la lisibilite texture, la contrainte plateforme et les premiers criteres d'acceptation. La sortie est une note de revue candidat.

### Scenario 3 - Controle Handoff Unity

Preparer le handoff comme si un autre reviewer Unity devait l'inspecter: nom, note source, format, echelle, orientation, pivot, bounds, materiaux, attentes texture et role scene. La sortie est un resultat de checklist oriente Unity.

### Scenario 4 - Revue Workflow Produit

Cadrer le meme candidat avec le langage Mob'ia / ccomf-unity: profil, job, artefact, etat, client et prochaine action. La sortie est un etat workflow lisible cote produit.

### Scenario 5 - Preparation LocalAssetFactory

Passer le candidat dans le langage de preparation locale: manifest, normalisation, resultat import attendu, statut de revue et prochaine action. La sortie est une note de readiness asset locale.
