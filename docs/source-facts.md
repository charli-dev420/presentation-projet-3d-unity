# Source Facts / Faits sources

[EN](#english) | [FR](#francais)

## English

### Confirmed Public Facts

**Dataset ReviewEval** is represented as the source-review app for visual dataset evaluation and exportable review notes. Its role in this showcase is to make source choice explainable before asset preparation.

**Splat Face / Splat Facade Baker** is represented as the main 3D/Unity direction. It focuses on lightweight facade and 2.5D asset candidates, with criteria for mobile use, visual readability, and Unity review.

**CodexUnity / CodexToUnity** is represented as the Codex / Unity / ComfyUI bridge for requests, manifests, candidates, and import checks.

**Mob'ia / ccomf-unity** is represented as the product surface for profiles, jobs, artifacts, review states, and clients.

**LocalAssetFactory** is represented as the local preparation and validation loop for asset candidates before Unity review.

### Interpretation

The showcase should be read as a product dossier for a pipeline family, not as a single finished application. The strongest product story is the path from source quality to Splat Face candidate to Unity-facing decision.

## Francais

### Faits Publics Confirmes

**Dataset ReviewEval** est presente comme app de revue source pour evaluation dataset visuelle et notes de revue exportables. Son role dans cette vitrine est de rendre le choix source explicable avant la preparation asset.

**Splat Face / Splat Facade Baker** est presente comme la direction 3D/Unity principale. Le projet se concentre sur candidats assets facade et 2.5D legers, avec criteres pour usage mobile, lisibilite visuelle et revue Unity.

**CodexUnity / CodexToUnity** est presente comme le pont Codex / Unity / ComfyUI pour demandes, manifests, candidats et controles import.

**Mob'ia / ccomf-unity** est presente comme la surface produit pour profils, jobs, artefacts, etats de revue et clients.

**LocalAssetFactory** est presente comme la boucle locale de preparation et validation de candidats assets avant revue Unity.

### Interpretation

La vitrine doit etre lue comme un dossier produit pour une famille de pipelines, pas comme une seule application finalisee. Le recit produit le plus fort est le chemin qualite source -> candidat Splat Face -> decision orientee Unity.
