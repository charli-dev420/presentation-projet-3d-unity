# Decision Pack / Dossier de decision

[FR](#francais) | [EN](#english)

## Francais

### Demo prioritaire recommandee

1. DatasetViewEval: petit batch public-safe, verdicts et rapport.
2. SFB: conversion maps/depth-card avec asset partageable.
3. Unity: import dans projet jetable, pivot/echelle/bounds visibles.
4. CodexToUnity/LocalAssetFactory: dry-run puis run controle si l'environnement est pret.

### Pour un financeur

- Financer les validations live Unity, Blender, ComfyUI.
- Financer une bibliotheque de cas d'usage publics ou sous NDA.
- Financer QA asset: bounds, triangles, textures, materials, profils.

### Pour un partenaire Unity

- Fournir contraintes projet et criteres de performance.
- Choisir un format cible: prefab, scene, depth-card, GLB, package.
- Exiger une preuve importable dans un projet jetable.

### Questions de decision

1. Quelle sortie asset est prioritaire?
2. Quelle preuve live doit etre fermee d'abord?
3. Quelles donnees peuvent etre publiees?
4. Quel format de demo rassure le plus: video, repo public, rapport, scene Unity?

## English

The recommended demo path is DatasetViewEval batch, SFB depth-card conversion, Unity import in a disposable project, then CodexToUnity/LocalAssetFactory controlled run when the environment is ready.
