## English

### Product Definition

CodexUnity / CodexToUnity is the bridge between AI-assisted work and Unity review. It gives the asset pipeline a language for requests, jobs, manifests, candidate outputs, import checks, and written review notes.

The value is the handoff. A file alone does not explain scale, pivot, materials, expected scene role, source decision, or acceptance criteria. CodexUnity / CodexToUnity is the place where those details become explicit.

### Who It Helps

It helps Unity integrators, tool builders, technical artists, AI workflow reviewers, and collaborators who need generated or prepared assets to arrive with enough context to be inspected.

### Workflow

1. Frame the Unity target and expected asset role.
2. Define job shape, manifest expectations, output names, and review criteria.
3. Run a controlled preparation path or dry path.
4. Check import behavior, bounds, scale, orientation, materials, naming, and scene usefulness.
5. Write the review note and pass the candidate to Splat Face, Mob'ia, or LocalAssetFactory context as needed.

### What This Repository Shows

This repo explains where CodexUnity / CodexToUnity fits in the 3D pipeline, what a Unity handoff should contain, and how it supports Splat Face without replacing product or technical-art review.

### Useful Support

Useful support includes manifest design, Unity importer review, handoff UX, ComfyUI workflow feedback, QA criteria, editor validation, and documentation.

### Workflow

4. Controler import, bounds, echelle, orientation, materiaux, nommage et utilite scene.
5. Ecrire la note de revue et passer le candidat au contexte Splat Face, Mob'ia ou LocalAssetFactory si necessaire.
